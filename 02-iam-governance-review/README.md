# IAM Governance Review

## Project Overview

This project presents an IAM governance review of a personal AWS cloud lab environment. The review focuses on IAM users, IAM user groups, permissions, MFA status, group-based access assignment, and CloudTrail evidence for identity-related activity.

The purpose of this project is to demonstrate Cloud Governance, Risk, and Compliance skills by reviewing access controls, identifying privilege risks, documenting IAM governance observations, and recommending least-privilege improvements.

## Environment Scope

The review covers the following IAM users and groups:

### IAM Users

- cloud-grc-admin
- grc-readonly-user
- grc-developer-user

### IAM User Groups

- CloudGRC-Admins
- CloudGRC-Developers
- CloudGRC-ReadOnly

## AWS Services Reviewed

- AWS Identity and Access Management
- AWS CloudTrail Event History

## Project Objectives

- Review IAM users in scope
- Review IAM user groups and attached policies
- Validate group-based access assignment
- Review MFA status for privileged access
- Identify excessive permission risks
- Document access governance findings
- Recommend least-privilege improvements
- Collect audit-style evidence using AWS screenshots and CloudTrail Event History

## Skills Demonstrated

- IAM governance
- Access control review
- Least privilege analysis
- Cloud security documentation
- Risk register development
- Audit evidence collection
- CloudTrail event review
- GRC-style findings and recommendations

## Disclaimer

This project was created in a personal AWS lab environment for learning and portfolio purposes. No real company data, production systems, or sensitive information were used.
