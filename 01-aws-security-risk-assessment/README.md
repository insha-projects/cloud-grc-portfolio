# AWS Security Risk Assessment

## Project Overview

This project presents a security and compliance assessment of a basic AWS cloud lab environment. The environment includes AWS account security, IAM users and groups, MFA, S3 bucket security, billing safeguards, and CloudTrail Event History.

The purpose of this project is to demonstrate Cloud Governance, Risk, and Compliance skills by identifying cloud assets, reviewing implemented security controls, documenting risks, collecting audit-style evidence, and providing recommendations for improvement.

## Environment Scope

The assessment covers the following areas:

- AWS account security
- IAM users and user groups
- Multi-factor authentication
- AWS billing and budget alerts
- Amazon S3 bucket security
- CloudTrail Event History
- Basic cloud governance documentation

## AWS Services Reviewed

- AWS Identity and Access Management
- Amazon S3
- AWS CloudTrail
- AWS Billing and Cost Management
- AWS Budgets

## Project Objectives

- Secure the AWS root user with MFA
- Create a separate IAM admin user for daily lab work
- Configure billing access and budget alerts
- Create IAM groups for access governance demonstration
- Create test users and assign them to appropriate groups
- Create and review an S3 bucket with public access blocked and default encryption enabled
- Review CloudTrail Event History for audit visibility
- Document assets, risks, controls, findings, and recommendations

## Skills Demonstrated

- Cloud GRC documentation
- AWS security fundamentals
- IAM governance
- Access control review
- Risk register development
- Security control assessment
- Audit evidence collection
- CloudTrail event review
- Cost governance awareness

## Evidence Collected

Evidence screenshots were collected for MFA configuration, IAM billing access, budget alerts, IAM groups, IAM users, S3 bucket security, and CloudTrail Event History.

## Disclaimer

This project was created in a personal AWS lab environment for learning and portfolio purposes. No real company data, production systems, or sensitive information were used.
